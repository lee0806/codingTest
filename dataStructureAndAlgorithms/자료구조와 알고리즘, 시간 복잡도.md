

**자료구조란?**

> 데이터를 어떤 방식으로 저장하고 관리할지에 대한 구조

예시
- 단순 자료구조 : 변수 배열(Array)
- 선형 자료구조 : 스택, 큐, 연결리스트, 해시 테이블
- 비선형 자료구조 : 트리(Tree), 그래프(Graph), 힙(Heep)

**왜 필요할까?**

문제를 효율적으로 해결하기 위해 데이터를 어떻게 다룰지 먼저 결정한다.
-> 자료구조 선택이 알고리즘의 효율을 좌우함

**알고리즘이란?**

> 문제를 해결하기 위한 절차적 방법 또는 논리

- 자료구조 위에서 실행
- 같은 자료구조라도 성능이 다른 여러 알고리즘이 존재한다.
- 자료구조를 보고 적절한 알고리즘을 선정할 수 있어야 함

## 시간 복잡도

- 특정 알고리즘이 어떤 문제를 해결하는 데 걸리는 시간

- **같은 알고리즘이라도 성능에 따라 실행시간이 다름**

- 코드에서의 성능이 영향을 끼치는 부분들을 보고 시간을 예측함

- **반복문을 보고 성능을 평가**

- Big-O : 최악의 경우의 수
- 배열의 길이가 n일때 n번만에 조건을 만족할때 : **O(n)** 으로 표기 -> y = x 그래프

- 시간 복잡도 순서
- O(1) -> O(logn) -> O(n) -> O(nlogn) -> O(n^2) -> O(2^n) -> O(n!)


**시간 복잡도란?**

> 알고리즘이 실행되는 데 걸리는 시간, 특정 알고리즘이 어떤 문제를 해결하는 데 걸리는 시간

- 입력 크기 n에 따라 성능을 예측
- 실행 횟수를 기준으로 측정 

**Big-O 표기법**

> 최악의 경우를 기준으로 표기

배열의 길이가 n일때 n번만에 조건을 만족하면 O(n)으로 표기

**시간 복잡도 순위**

- O(1) : 입력 크기에 상관없이 복잡도는 일정
- O(log n) : 매번 반으로 줄어듦
- O(n) : 크기에 비례
- O(n log n) : 최적 정렬 알고리즘
- O(n^2) : 완전 탐색
- O(2^n) : 크기가 2배씩 증가
- O(n!) : 순열 문제

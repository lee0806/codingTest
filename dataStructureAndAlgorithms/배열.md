**배열이란?**

> 동일한 자료형을 가진 데이터를 연속된 메모리 공간에 저장한 자료구조

- 메모리 상에서 한 줄로 이어져 있음
- 배열의 길이를 미리 알아야 함 (크기 고정)

**배열의 접근이 왜 빠를까? O(1)**

- 컴퓨터는 배열의 시작 주소를 알고 있음
- 인덱스로 접근 시 : 시작 주소 + (자료형의 크기 * 인덱스) 계산만 하면됨
- 따라서 즉시 접근이 가능해 O(1)의 복잡도를 가짐
- 따라서 배열은 읽기 / 쓰기 속도가 매우 빠름

**하지만 삽입 / 삭제가 느림**

배열은 연속된 메모리이기 때문에 중간에 값을 넣거나 빼면 새로운 메모리를 할당해야하므로 모든 데이터가 이동해야됨

따라서 시간 복잡도는 O(n)

```
기존 메모리 공간이 꽉 찼을때 데이터를 추가할 경우

- 더 큰 공간을 찾고
- 기존 데이터를 복사하고
- 이전 메모리를 해제 해야함
```

그렇다고 큰 배열을 미리 잡아도, 더 작게 잡아도 안됨
-> 메모리의 낭비와 확장하는데 또 메모리를 사용하기 때문

**그럼 자바스크립트의 배열은?**

자바스크립트는 정적인 배열이 아니라 리스트 형태를 가진다(연결된 하나의 배열이 아닌, 다음 위치를 알고 있는 리스트임).

- 자료형이 달라도 배열이 가능해짐
- 크기를 자유롭게 변경하거나 추가할 수 있음
- 연속된 메모리 보장이 없음

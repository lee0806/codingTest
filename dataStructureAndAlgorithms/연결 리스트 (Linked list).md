**Linked list가 왜 등장했는가?**

배열은 크기가 고정되고, 삽입 / 삭제가 느린점이 있음
-> 이를 보안하기 위해 데이터를 연속된 메모리에 저장하지 않는 자료구조가 필요해짐

**Linked list의 구조**

연결 리스트는 노드(Node)라는 단위들로 이루어져 있음

각 노드는 : 
	data(값 저장)
	next 포인터(다음 노드 주소 저장)

```
[data | next] -> [data | next] -> [data | next]
```

**Linked list의 장점**

- 메모리를 연속해서 할당할 필요가 없음
- 삽입 / 삭제가 빠름
- 크기를 유동적으로 변경할 수 있음

**Linked list의 단점**

- 인덱스 접근이 느림 (이전 노드를 따라가야 하기 때문에 O(n))
- 추가 포인터의 공간이 필요함 (next 저장 공간 필요)
- 캐시 효율이 떨어짐 (메모리가 분산되어 있어서)

|**연산**|**배열(Array)**|**연결 리스트(Linked List)**|
|---|---|---|
|접근(access)|**O(1)**|**O(n)**|
|삽입/삭제 (앞/중간)|**O(n)**|**O(1)**|
|검색(search)|**O(n)**|**O(n)**|
